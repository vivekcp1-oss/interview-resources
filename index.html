Artificial Intelligence (AI)

The broad concept of machines executing tasks in a way we consider "smart." This can include simple rule-based logic.

Machine Learning (ML)

The statistical application of AI. Instead of rules, the system probabilistically determines outcomes based on historical patterns.

Deep Learning (DL)

A specialized sub-field utilizing multi-layered neural networks to handle complex, unstructured data like speech or imagery.

Core Types of Machine Learning

Categorized by the learning signal: whether labels are provided, rewards are given, or structure is discovered.

🎯

Supervised Learning

Requires labeled data. The model learns from a known "answer key" to map inputs to specific targets.

model.fit(X, y)

Case: Fraud Detection

🔍

Unsupervised Learning

Uses unlabeled data to discover hidden patterns, groupings, or structural relationships without a target.

KMeans().fit(X)

Case: Customer Segments

🎮

Reinforcement Learning

Learning via trial and error. An agent interacts with an environment to maximize cumulative reward signals.

agent.step(action)

Case: Game AI

Learning Characteristics

Supervised models excel at prediction accuracy given historical labels, while Unsupervised models are the primary choice for exploratory data analysis.

Supervised: High Label Dependency
Unsupervised: Pattern Discovery Focus

The 80/20 Rule: Data Splitting

Partitioning data prevents "overfitting"—a state where the model memorizes noise instead of learning generalizable patterns.

GIGO Principle

"Garbage In, Garbage Out." A model is only as good as the data cleaning performed. Removing outliers and handling missing values is the priority.

Fatal Error

Data Leakage: Unintentionally exposing test-set information during training destroys real-world reliability.

Training vs. Inference Phases

Modern ML systems split workloads between heavy-duty compute for learning and low-latency response for users.

Phase A: Training

Teaching the Engine (Batch Process)

Adjusting millions of internal weights using historical data. Requires massive GPU/TPU power and happens offline.

Phase B: Inference

Serving the Prediction (Real-Time)

Processing live user input through the static trained model. Requires millisecond latency and optimized deployment.

"Netflix trains on months of history (Training) to suggest a movie in seconds (Inference)."

Advanced Vocabulary

BB

Black Box

Highly complex models (DL) where decision paths are not human-interpretable.

F/L

Feature/Label

Input variables vs. the target outcome we aim to predict.

CI

Class Imbalance

When one class dominates the dataset, causing the model to lean towards majority bias.

SN

Scaling

Normalizing features so magnitude differences don't break the statistical math.
